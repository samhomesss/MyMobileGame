UI 작업을 할때 Canvas 에서 Canvas Scaler에 UI Scale Mode를 Scale With Screen Size로 변환 하는것이 좋다.
-> 해당 방식을 사용하는 이유는 내가 기종이 바뀌거나 해당 기기가 바뀌었을때 마다 UI의 스케일이 바뀌면 안되는 것이기 때문에 
   해당 방식을 처리함으로써 어떤 기기로 변환 하든 스케일을 고정 할 수 있다.

Canvas Scaler 의 Screen Match Mode의 기능 -> 해당 스케일에 맞지 않은 기종이면 어떻게 대응 할 것 인가?

UI를 배치 하다 보면 클릭이 되어야 하는 오브젝트와 클릭이 되지 말아야 할 UI 두가지가 존재 하는데 이를 판정 하는 것 이 RayCast 시리즈 이다 
클릭이 되어야 하는 오브젝트의 경우에는 Raycast Target을 켜주고 클릭이 되지 말아야 하는 경우의 오브젝트의 경우는 Raycast Tartget을 꺼줌으로 써 
UI관련 최적화를 할 수 있다.

해당 DisplayText 택스트의 경우에는 해당 텍스트를 눌렀을때 이벤트가 실행 되는것을 만들지 않을 것 이기 때문에 해당 텍스트에 대한 이벤트를 
사용하지 않을 것이기에 Raycast Target 을 꺼준다. -> 해당 오브젝트로 이벤트가 발생 하는 상황이라면 Raycast Target을 켜주는 것이 맞다.

현재 프로젝트에서 해당 Touch 기능을 담당 하고 있는 것이 보이지는 않지만 백 그라운드 에 있는 Image 이다.

만약 내가 아트적으로 뭐 UI를 만들고 이벤트를 실행 시키는 오브젝트를 만들었다면 그에 대응 되는 UI를 만드는 것이 맞다.
